# Standings Feature - Implementation Guide

## Architecture Overview

The standings feature uses a **separate collection** approach rather than embedding data in team documents. This provides:

- ✅ Single source of truth (one document per league/season)
- ✅ No data redundancy (team details stay in Team collection)
- ✅ Efficient queries (entire table in one query)
- ✅ Easy updates (update once per league vs 20+ team docs)
- ✅ Multi-competition support (teams in multiple leagues/cups)

## Database Schema

### Standing Model

Location: `backend/models/Standing.js`

```javascript
{
  // League/Competition identifiers
  league_id: Number,      // Sportmonks league ID (e.g., 8 = Premier League)
  league_name: String,
  season_id: Number,      // Sportmonks season ID
  season_name: String,    // e.g., "2025/2026"
  
  // Stage info (for cups with multiple stages)
  stage_id: Number,
  stage_name: String,     // e.g., "Regular Season", "Quarter Finals"
  
  // Round info
  round_id: Number,
  round_name: String,
  
  // Type
  is_cup: Boolean,
  
  // The standings table (array of team standings)
  table: [{
    position: Number,           // 1, 2, 3, etc.
    participant_id: Number,     // Sportmonks team ID
    team_id: ObjectId,          // Reference to our Team model (optional)
    points: Number,
    trend: String,              // 'up', 'down', 'equal'
    
    // Statistics
    played: Number,
    won: Number,
    drawn: Number,
    lost: Number,
    goals_for: Number,
    goals_against: Number,
    goal_difference: Number,
    
    // Recent form
    form: [String],             // ['W', 'W', 'D', 'L', 'W'] - last 5 matches
    
    // Rule (for promoted/relegated zones)
    standing_rule_id: Number
  }],
  
  // Metadata
  last_updated: Date,
  updated_at: Date,
  createdAt: Date            // Auto-generated by timestamps
}
```

### Indexes

- `{ league_id: 1, season_id: 1 }` - Query by league and season
- `{ season_id: 1, stage_id: 1 }` - Query by season and stage
- `{ 'table.participant_id': 1 }` - Find team standings across competitions

## API Endpoints

### Public Endpoints (GET)

#### Get League Standings
```bash
GET /api/standings/league/:leagueId

# Example
GET /api/standings/league/8
```

Returns full standings table for most recent season of the league.

#### Get Team Standings (All Competitions)
```bash
GET /api/standings/team/:participantId

# Example
GET /api/standings/team/19  # Arsenal
```

Returns all standings where this team appears (multiple leagues/cups).

#### Get Team Position in Specific League
```bash
GET /api/standings/league/:leagueId/team/:participantId

# Example
GET /api/standings/league/8/team/19  # Arsenal in Premier League
```

Returns just the team's standing entry for that league.

### Admin Endpoints (POST - requires API key)

#### Sync League Standings
```bash
POST /api/standings/sync/league/:leagueId
Header: x-api-key: YOUR_ADMIN_KEY

# Example
POST /api/standings/sync/league/8
```

Fetches current season standings from Sportmonks and saves to database.

#### Sync Season Standings
```bash
POST /api/standings/sync/season/:seasonId
Header: x-api-key: YOUR_ADMIN_KEY

# Example
POST /api/standings/sync/season/25583
```

Fetches standings for a specific season.

#### Batch Sync Multiple Leagues
```bash
POST /api/standings/sync/batch
Header: x-api-key: YOUR_ADMIN_KEY
Content-Type: application/json

{
  "league_ids": [8, 9, 24],  // Premier League, Championship, FA Cup
  "delay_ms": 500            // Delay between requests (rate limiting)
}
```

Syncs multiple leagues in one call.

## Usage Examples

### Backend Service

```javascript
const {
  syncStandingsByLeague,
  getStandingsForLeague,
  getTeamPositionInLeague
} = require('./services/standingsService');

// Sync Premier League standings
await syncStandingsByLeague(8);

// Get standings from database
const standings = await getStandingsForLeague(8);

// Display table
standings.table.forEach(entry => {
  console.log(`${entry.position}. ${entry.points} pts - Form: ${entry.form.join('-')}`);
});

// Get Arsenal's position
const arsenalPos = await getTeamPositionInLeague(8, 19);
console.log(`Arsenal: ${arsenalPos.position}th with ${arsenalPos.points} points`);
```

### Frontend API Calls

```javascript
// src/api.js
export async function getLeagueStandings(leagueId) {
  return req(`/api/standings/league/${leagueId}`);
}

export async function getTeamStandings(participantId) {
  return req(`/api/standings/team/${participantId}`);
}

// Usage in component
const standings = await getLeagueStandings(8);  // Premier League

standings.data.table.forEach(entry => {
  // Render table row
  // You have: position, points, played, won, drawn, lost, 
  //           goals_for, goals_against, goal_difference, form, trend
});
```

### Cron Job (Daily Update)

Add to `backend/cron/index.js`:

```javascript
const { syncMultipleLeagues } = require('../services/standingsService');

const LEAGUES_TO_SYNC = [
  8,   // Premier League
  9,   // Championship
  24,  // FA Cup
  271, // League One
  272  // League Two
];

// Run every day at 2 AM
cron.schedule('0 2 * * *', async () => {
  console.log('[cron] Syncing standings...');
  
  try {
    const results = await syncMultipleLeagues(LEAGUES_TO_SYNC, 1000);
    const successful = results.filter(r => r.success).length;
    console.log(`[cron] ✅ Synced ${successful}/${LEAGUES_TO_SYNC.length} standings`);
  } catch (error) {
    console.error('[cron] ❌ Standings sync failed:', error);
  }
});
```

## Data Mapping

The API returns statistics with `type_id` codes. These are mapped in `utils/standingsParser.js`:

```javascript
const STANDING_TYPE_MAP = {
  187: 'points',
  185: 'played',
  133: 'goals_for',
  134: 'goals_against',
  129: 'home_won',
  130: 'away_won',
  // ... etc
};
```

The service automatically:
1. Parses the `details` array to extract stats
2. Calculates derived values (total won = home_won + away_won)
3. Extracts form from the `form` array
4. Links to our Team documents via `team_id`

## Frontend Integration

### Example React Component

```jsx
import { useState, useEffect } from 'react';
import { getLeagueStandings } from '../api';

function StandingsTable({ leagueId }) {
  const [standings, setStandings] = useState(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    async function fetchStandings() {
      try {
        const data = await getLeagueStandings(leagueId);
        setStandings(data.data);
      } catch (error) {
        console.error('Failed to load standings:', error);
      } finally {
        setLoading(false);
      }
    }
    
    fetchStandings();
  }, [leagueId]);
  
  if (loading) return <div>Loading...</div>;
  if (!standings) return <div>No standings available</div>;
  
  return (
    <div>
      <h2>{standings.league_name} - {standings.season_name}</h2>
      <table>
        <thead>
          <tr>
            <th>Pos</th>
            <th>Team</th>
            <th>P</th>
            <th>W</th>
            <th>D</th>
            <th>L</th>
            <th>GF</th>
            <th>GA</th>
            <th>GD</th>
            <th>Pts</th>
            <th>Form</th>
          </tr>
        </thead>
        <tbody>
          {standings.table.map((entry) => (
            <tr key={entry.participant_id}>
              <td>{entry.position}</td>
              <td>{/* Lookup team name from Team collection */}</td>
              <td>{entry.played}</td>
              <td>{entry.won}</td>
              <td>{entry.drawn}</td>
              <td>{entry.lost}</td>
              <td>{entry.goals_for}</td>
              <td>{entry.goals_against}</td>
              <td>{entry.goal_difference}</td>
              <td>{entry.points}</td>
              <td>{entry.form.join('-')}</td>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
}
```

## Notes

- **No live standings**: The live endpoint is not available in your Sportmonks plan
- **Rate limiting**: Use delays between requests when batch syncing
- **Team names**: You'll need to join with Team collection to get team names for display
- **Promotion zones**: Use `standing_rule_id` to identify promoted/relegated zones
- **Cups**: Cup competitions may have different structure (stages/groups)
